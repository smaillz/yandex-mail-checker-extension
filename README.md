# Watcher for Yandex Mail

Расширение для Chrome (Manifest V3), которое показывает счётчик непрочитанных писем Яндекс.Почты на иконке панели инструментов и позволяет быстро открыть почту.

## Возможности

- Счётчик непрочитанных писем на иконке (бейдж), цветная/серая иконка для состояний «есть почта» / «не авторизован».
- Периодическая фоновая проверка через `chrome.alarms` с настраиваемым интервалом (или «никогда»).
- Индикатор проверки (синий бейдж `…`) и понятные подсказки‑тултипы.
- Поведение по клику: меню‑попап **или** одиночный клик = открыть почту / двойной = проверить сейчас.
- Гибкое открытие почты: переиспользование существующей вкладки, текущая/новая/фоновая вкладка.
- Поддержка нескольких доменов Яндекса: `yandex.com`, `.by`, `.kz`, `.ru`, `.com.tr`, `.ua`.
- Локализация интерфейса: язык выбирается **в настройках расширения** (Авто / English / Русский), а не зависит от языка браузера.

## Требования

- Chrome **94+** (используется service worker в виде ES‑модуля; поддержка появилась в Chrome 91, минимум поднят до 94).

## Установка (режим разработчика)

1. Открой `chrome://extensions`.
2. Включи **Режим разработчика** (Developer mode) в правом верхнем углу.
3. Нажми **Загрузить распакованное расширение** (Load unpacked) и выбери папку проекта.
4. Иконка появится на панели инструментов. Авторизуйся в Яндекс.Почте в этом же браузере.

После изменений в коде нажми «Обновить» (↻) на карточке расширения в `chrome://extensions`.

## Настройки

Открываются через меню‑попап → «Настройки…» или со страницы `chrome://extensions`.

- **Язык** — язык интерфейса (Авто / English / Русский).
- **Почтовый сайт** — выбор домена Яндекса.
- **Проверка почты** — считать непрочитанные только во «Входящих» или с учётом своих папок.
- **Автопроверка** — интервал фоновой проверки (от 1 минуты до «никогда»).
- **Иконка на панели** — показывать ли счётчик; меню по клику или одиночный/двойной клик.
- **Открытие почты** — сбрасывать ли счётчик при открытии, переиспользовать вкладку, куда открывать письмо.

> Примечание: при включённой опции «Сбрасывать счётчик при открытии почты» бейдж очищается сразу после открытия почты; реальное число вернётся при следующей авто‑проверке, если письма остались непрочитанными.

## Структура проекта

```
manifest.json            # Манифест MV3 (service worker, action, host_permissions)
_locales/                # Переводы (en, ru) в формате chrome.i18n messages.json
icons/                   # Иконки расширения (source.png — исходник)
css/                     # Стили страниц (options.css, popup.css)
html/                    # Разметка страниц (options.html, popup.html)
js/
  background.js          # Service worker: проверка почты, иконка, расписание, открытие
  edition.js             # URL Яндекса и парсер HTML lite-инбокса (analyzeHTML)
  i18n.js                # Загрузчик локали по выбору пользователя + локализация страниц
  options.js             # Логика страницы настроек
  popup.js               # Логика меню‑попапа
```

## Как это работает

- **Service worker** (`background.js`) по будильнику (`chrome.alarms`) запрашивает страницу lite‑инбокса (`fetch` с `credentials: "include"`), парсит её через `analyzeHTML` и обновляет иконку/бейдж/тултип.
- Иконки рисуются из PNG через `OffscreenCanvas` в `ImageData` — это надёжно работает в service worker (в отличие от `setIcon({ path })`).
- **Локализация**: вместо `chrome.i18n` (который жёстко привязан к языку браузера) используется собственный загрузчик `_locales/<lang>/messages.json`, где `<lang>` берётся из настроек.
- Настройки хранятся в `chrome.storage.local` под ключом `preference`; страница настроек шлёт сообщение `prefsUpdated`, по которому worker пересоздаёт будильник и сразу проверяет почту.

## Ограничения

- Парсер `analyzeHTML` рассчитан на текущую вёрстку lite‑версии Яндекс.Почты и может потребовать правок при изменении разметки.
- Требуется активная сессия (авторизация) в Яндекс.Почте в том же профиле браузера.
